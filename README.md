# NSU Banner Table Documentation

Built with [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).

## Local Development

```bash
pip install -r requirements.txt
python tools/page-generator/generate.py
mkdocs serve
```

Site runs at `http://127.0.0.1:8000`. Pushing to `main` auto-deploys to GitHub Pages.

## Project Structure

```
main.py                     ← interactive CLI tool and main entry point for users
table-definitions/          ← source of truth (what you edit)
├── SPRIDEN/
│   ├── SPRIDEN.json        ← table metadata
│   ├── SPRIDEN.dat         ← lookup values (definition tables only)
│   └── sql/
│       └── find-name.sql   ← queries
tools/
├── info-converter/         ← converts .info files to JSON
│   ├── convert.py
│   ├── info-files/         ← drop .info files here
│   └── converted/          ← JSON output
└── page-generator/
    └── generate.py         ← builds markdown from JSON
docs/                       ← generated (don't edit directly)
```

## CLI Tool Usage
```
python main.py                  # Launch interactive REPL
python main.py convert [TABLE]  # Convert .info files to JSON (all, or one table)
python main.py update-refs      # Update table references
python main.py generate         # Generate MkDocs pages
python main.py all [TABLE]      # Run convert, update-refs, and generate in sequence (all, or one table)

# MKDocs commands:
python main.py status           # Show pipeline gaps (unconverted/unplaced tables)
python main.py build            # Build the MkDocs site (--strict)
python main.py serve            # Serve the MkDocs site locally
```

## JSON Reference

```json
{
  "name": "SPRIDEN",
  "description": "Person Identification/Name Repeating Table",
  "type": "data",
  "tags": ["core", "auth"],
  "definitions_headers": ["Code", "Description"],
  "columns": [
    {
      "name": "SPRIDEN_PIDM",
      "type": "NUMBER(8,0)",
      "description": "Internal identification number.",
      "definition_table": ""
    }
  ],
  "queries": [
    {
      "file": "find-name.sql",
      "name": "Find Name by ID",
      "description": "Look up a person's name by their PIDM."
    }
  ]
}
```

| Field | Required | Description |
| --- | --- | --- |
| `name` | Yes | Table name (auto-filled by converter) |
| `description` | Yes | What the table is for (auto-filled by converter) |
| `type` | Yes | `data` or `definition` |
| `tags` | Yes | Array of tags for categorization and search |
| `definitions_headers` | No | Column headers for `.dat` file (defaults to `["Code", "Description"]`) |
| `columns` | Yes | Array of column objects (auto-filled by converter) |
| `columns[].definition_table` | No | Name of a definition table this column references |
| `queries` | Yes | Array of query objects (empty array if none) |

## .info File Format

Generated by running `INFO <TABLE_NAME>;` in the SQL editor. Copy the entire output including the table header, columns, and indexes sections. Save as `<TABLE_NAME>.info`.

## .dat File Format

Tab or multi-space delimited, no header row. Paste directly from database output:

> [!IMPORTANT]
> If you are using a definition file, you should also be filling out the `definition_headers` part of the json.


```
AGS     Associate of General Studies
BA      Bachelor of Arts
BS      Bachelor of Science
```
